\subsection{Préparation des Test Cases}

La premier tâche à faire c'est d'étudier les fonctionnalités principale de l'application et comprendre son rôle puis réaliser les "Test Case" pour cette application  qui permet au Biologiste de valider et consulter les résultats d'une analyse médicale.\\

	\begin{center}
 \includegraphics[scale=0.6]{image0.png} \end{center}

	\caption{Le Guide Code Couleurs de l'application ProLab Mobile 5}
	\label{imag4}
\end{figure}

Ensuite, faut comprendre c'est quoi la structure d'une Test Case et les éléments nécessaire et ce sont :\\
\medskip
\begin{itemize}
\item \textbf{}"Test Setup/Device" c'est où déroule le test soit dans une appareil mobile avec un test manuel ou soit dans une Emulator/Simulator avec un test automatique.
\medskip

\item \textbf{}"Test Case ID" et "Test Case Description" pour bien préciser le sujet qu'on va tester.
\medskip

\item \textbf{}Test Steps, c'est les étapes a suivre pour vérifier ce Test.
\medskip

\item \textbf{}"Pre-Conditions", c'est les conditions nécessaires qu'on est besoin d'avoir avant de faire ce Test.
\medskip

\item \textbf{}"Test Data", au cas ou on est besoin de Data dans un Test et cette colonne peut-être affiché dans un fichier entier si la Data est trop volumineux.
\medskip

\item \textbf{}"Expected Result" et "Actual Result" sont les plus importantes parties d'un Test Case vu qu'on est besoin de connaitre c'est quoi la résultat attendu et si elle matche le résultat données.
\medskip

\item \textbf{}"Status" Si le résultat attendu et même que le résultat donné alors c'est une test sucées sinon c'est un échec.
\medskip

\item \textbf{}"Executed By" c'est celui qui a fait/écrit le Test.
\medskip

\item \textbf{}"Executed Date" est la date d'exécution de cette Test.
\end{itemize}
